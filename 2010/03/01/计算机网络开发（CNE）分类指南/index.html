计算机网络开发（CNE）分类指南

发表于 2010-03-01 | 分类于 绝密

题目: 计算机网络开发（CNE）分类指南
发行日期: 2014-10-10
文件日期: 2010-03-01
描述: 2010年NSA分类指南概述了该机构黑客活动各方面的敏感性。
文件类型: 绝密
关键字: 计算机网络利用（CNE）;NSA/CSS或TAO;远程颠覆;物理颠覆;ECI;

1.（U）NSA/CSS或TAO执行计算机网络利用（CNE）

2.（S // REL）NSA/CSS或TAO，作为CNE操作的一部分，执行远程颠覆

3.（S // SI // REL）NSA/CSS或TAO使用的特定远程subversion方法的标识，包括：

  • 端点访问，利用或操作
  • 网上访问，利用或操作
  • 软件植入访问，利用或操作
  • 访问或利用其他数据

4.（S // SI // REL）作为CNE操作的一部分，NSA/CSS或TAO执行物理颠覆的事实包括：

  • 关闭访问的可能性，利用或操作
  • 网外启用，利用或操作
  • 供应链启用，利用或干预操作
  • 硬件植入启用，利用或操作需要ECI保护。

5.（U）任何特定ECI名称或三元组与NSA/CSS，ECI，SIGINT，intelligence

6.（U）特定TAO的ECI名称或三元组与CNE和TAO的关联

7.（U）当没有ECI与TAO之间没有关联，特定个人被清除特定TAO的ECI名

8.（U）NSA/CSS或TAO为外国情报收集进行CNE的事实

9.（U）NSA/CSS或TAO，作为CNE运营的一部分，执行CNE以支持美国政府CNA工作

10.（U）NSA/CSS或TAO作为CNE运营的一部分，训练，装备和组织美国密码系统，以支持其客户的CNE，CNA和CND要求

11.（U）NSA/CSS或TAO，作为CNE运营的一部分，提供与CNO相关的军事目标支持

12.（U）作为CNE行动的一部分，NSA/CSS或TAO提供情报收益/损失评估，以响应战斗指挥官（COCOM）CNO

13.（U）NSA/CSS或TAO作为CNE运营的一部分，开发并支持分析建模和模拟技术，以支持CNE/CNA工作

14.（U）作为CNE运营的一部分，NSA/CSS或TAO的目标，收集和处理计算机，计算机网络和计算机到计算机（C2C）通信，而不参考特定的操作，活动或目标

15.（S // SI // REL）作为CNE操作的一部分，NSA或TAO针对，收集和处理特定计算机协议（例如电子邮件，即时消息，文件传输协议）

16.（S // SI // REL）作为CNE操作的一部分，NSA/CSS或TAO将代码远程引入目标计算机网络以促进外国情报收集

17.（TS // SI // REL）作为CNE运营的一部分，NSA/CSS或TAO进行网外现场操作以开发，部署，利用或维护侵入式访问，而无需进一步详细说明

18.（S // SI // REL）作为CNE运营的一部分，NSA/CSS或TAO在NSA/CSS设施以外的特定地点开展网外活动

19.（U）TAO项目名称，与CNE或TAO联合，没有放大细节

20.（C / / REL）NSA/CSS或TAO，作为CNE运营的一部分，与第二方合作伙伴合作开展CNE活动

21.（C // REL）NSA/CSS或TAO作为CNE运营的一部分，与特定的第二方合作伙伴就特定ECI进行合作

22.（C // REL）NSA/CSS或TAO与第二方合作伙伴之间CNE合作的详细信息

23.（S // REL）NSA/CSS或TAO作为CNE运营的一部分，与未指明的第三方合作伙伴合作，支持和开展CNE活动

24.（S // REL）作为CNE运营的一部分，NSA/CSS或TAO与指定的第三方合作伙伴合作，支持和开展CNE活动

25.（U // FOUO）作为CNE运营的一部分，NSA/CSS或TAO与特定的美国政府/IC实体合作的事实

26.（C // REL）NSA/CSS或TAO作为CNE运营的一部分，与特定的美国政府/IC实体就特定ECf保密协作

27.（U）CNE工具的存在，没有进一步的细节/背景

28.（U）CNE工具的封面名称，没有详细信息

29.（S // SI // REL）当与远程颠覆相关联时，有关CNE工具的细节/描述，包括：

  • 特定类型（即硬件/软件等）
  • 目的
  • 能力
  • 隐藏技术
  • 电子签名
  • 上述结合

30.（S // SI // REL）当与物理颠覆相关联时，有关CNE工具的细节/描述，包括：

  • 特定类型（即硬件/软件等）
  • 目的
  • 能力
  • 隐藏技术
  • 电子签名
  • 上述结合

31.（U // FOUO）有关特定软件漏洞的技术细节，如公开已知，并被用于CNE活动

32.（S // SI / / REL）有关特定软件漏洞的技术细节，如果不是公开的，可以用于CNE活动

33.（U）NSA/CSS或TAO，作为CNE运营的一部分，针对特定国家或国际组织

34.（S // SI // REL）关于网外操作（即物理颠覆活动）的封面名称与放大细节（例如，特定电子元件，系统，其主机设施等）的关联

35.（S // REL）网上操作（即远程颠覆活动）的封面名称与放大细节的关联（例如，特定的电子元件，系统，其主机设施等）

36.（S // SI // REL）CNE活动的个别详细信息，例如：

  • 包括预期目标网络和/或设备的目标信息
  • 目标漏洞
  • 目标基础设施

37.（TS // SI // REL）NSA/CSS或TAO作为CNE操作的一部分，试图利用或成功利用特定漏洞（例如，在防火墙，操作系统，软件应用程序中）等，以及目标的IT /计算机结构中的特定实体或设施

38.（S // SI // REL）与目标的IT/通信系统上的美国硬件或软件植入物和位置（例如，特定组织和互联网协议设备/地址等）的描述相关的事实

39.（S // SI // REL）与确切时间，位置，参与者，网外或网上操作，CNE命令，控制和数据泄漏工具/功能和位置，用于利用或维护对目标的IT/计算机结构的侵入访问

40.（S // SI // REL）CNE活动各个方面的细节组合，允许特定目标采取特定的反措施，例如：

  • 特定目标网络或设备和
  • 特定能力，工具或用于利用脆弱性的技术

41.（TS // SI // REL）NSA/CSS（或TAO）通过CNE活动获取加密使能信息的事实

定义：
（U）计算机网络利用（CNE）：从收集数据的情报收集和启用操作目标或对手自动信息系统（A1S）或网络。
（U）计算机网络攻击（CNA）：业务操作，扰乱，拒绝，退化，或破坏信息的居民在计算机和计算机网络中，或计算机和网络本身。
（U）计算机网络防御（CND）：努力防范其他人的CNO，特别是针对美国和盟国的计算机和网络。
（U）计算机网络运营（CNO）：CNE，CNA和CND。
（U）信息业务（IO）：为保护自己的信息而采取的影响对手信息和信息系统的行动 和信息系统。IO是一种整合策略。

（S // S1 // REL）Intrusive Access：指涉及远程操作，硬件/软件修改的CNE操作或者感测计算机设备或系统中的环境变化，或偶尔感测容纳系统的设施 。
（S // SI // REL）网外行动：指为支持CNE活动而开展的人员秘密或秘密的实地活动。
（S // SI // REL）物理颠覆：对设备或主机设施进行物理访问的颠覆。有时用于表示物理颠覆的其他术语是：

  • 密切访问启用，利用或操作
  • 网外启用，剥削或操作
  • 供应链的启用，利用或运营
  • 硬件植入启用，利用或操作
    （S // SI // REL）远程颠覆：在没有物理访问设备或主机设施的情况下进行颠覆;有时用于暗示远程颠覆的其他术语是：
  • 计算机网络利用
  • 端点访问，利用或操作;
  • 网上访问，利用或操作;
  • 软件植入访问，利用或操作
  • 访问或利用静态数据
    （S // S1 // REL）供应链运营：专注于修改目标供应链中设备的阻截活动 。
