CNE出现在CT10状态报告中

发表于 2015-01-17 | 分类于 绝密

题目: CNE出现在CT10状态报告中
发行日期: 2015-01-17
文件日期: 未知
描述: 这篇未注明日期的NSA论文描述了一个识别和处理第三方攻击计算机数据的项目。
文件类型: 绝密
关键字: CT10数据模型，GHOSTWolf

执行摘要
（S // SI // REL USA，FVEY）本文讨论的工作自2008年4月1日起完成将CNE数据纳入CT10试点项目。CT10在这方面投入了两个方面。首先，是将活动的SIGINT收集并警告到CT10。其次，设计了当前CT10数据模型的扩展，以支持CNE数据的进一步分析开发。由于这项工作，CT10现在支持警告建立新的TAO存在点。关于某些开发方法的互联网。此外，这为内部和外部合作伙伴铺平了道路，为CT10 CNE数据模型扩展的持续发展铺平了道路 。
（S // SI // REL USA，FVEY）本文末尾包含了具体的结论和建议。但是，本文中的大部分信息都有三个关键点：
•（U // FOUO）对CT10数据采集工作的优先支持。
•（U // FOUO）进一步建立DNI/CNE分析和开发团队。
•（S // SI // REL USA，FVEY）制定在CT10中使用CNE数据的战略愿景。
（S // SI // REL USA，FVEY）处理CNE数据的第一步是识别和获取CNE数据流以将此信息路由到CT10。这项工作提出了自己独特的挑战，如下一节所述。

当前状态
（S // SI // REL USA，FVEY）CT10正在摄取单个CNE数据流，特别是FOXACID日志文件。这被CT10客户GHOSTWolf确定为首要任务 ，因此它被选为第一个追求的CNE数据流。在整个NSA中映射FOXACID数据流后，CT10从2008年2月中旬开始，从协议利用开始建立了这一数据流。
（S // SI // REL USA，FVEY）因为CNE数据模型扩展尚未实现此时，处理FOXACID日志的唯一方法是将信息拟合到当前的CT10数据模型中。这主要是因为当前模型的自然灵活性。在CT10摄取系统中定义新的“事件类型”很容易，并且从每个FOXACID事件中提取所有选择器来填充事件。当前模型设计的另一个好处是，为一个新的选择器类型定义变得容易植入ID并轻松将其集成到系统中。这允许将植入物ID添加到监视列表和目标选择器，以便CT10用户可以跟踪与定义的一组植入物相关联的任何事件。因此，如果CT10获得了包含植入回调信息的数据流，那么很容易将其添加到当前数据模型中，从而为分析人员提供跟踪通过FOXACID部署的植入物的附加功能。
（S // SI // REL USA，FVEY）由于这项工作，CT10系统现在通过一套警报技术提醒FOXACID开发活动。这包括GeoT中的地理位置倾斜，以及通过Agent Logic（IRC和电子邮件）和“空间仪表板”发出的警报。
（S // SI // REL USA，FVEY）从开发FOXACID摄取代码得出的一个结论是，当前的CT10数据模型可以支持比以前认为的更多的CNE数据 。当新的CNE流引入系统时，如果仍未实现扩展，则可以轻松地将它们添加到当前数据模型中 。

TAO数据流采集的近期和长期目标
（S // SI // REL USA，FVEY）关于CNE数据流入CT10的近期目标是确定额外的CNE数据流FOXACID数据。根据GHOSTWolf的要求，下一步将是获取植入回调数据流。这将允许分析追踪他们的时间植入者正在回复监听岗位上的任务和渗透。有问题的植入物是VALIDATOR，OLYUMPUS，UNITEDRAKE和STRAITBIZZARE。
（S // SI // REL USA，FVEY）建立此数据流后，下一步就是从中收集数据植入物。这包括任何exfiltrated文件和植入插件结果。通过每个新的数据流，CT10分析师将更全面地了解与其目标相关的CNE活动。但是，这些类型的数据自然不适合当前的数据模型，这就是提出的扩展如此重要的原因。
（S // SI // REL USA，FVEY）此外，TAO还没有创建其他对CT分析师有用的CNE数据集 。GOLLUM是一种合作植入物，是GHOSTWolf已经表明可以摄入CT10的一种数据集 。RADIUS日志（ISP拨打客户记录）也是一个很好的信息来源，并在DNR和DNI数据集之间创建自然链接。

挑战/未来之路
（S // SI // REL USA，FVEY）从TAO获得更多CNE数据存在许多挑战。首先，鉴于TAO数据流的复杂性，定位最终数据所有者具有挑战性。尽管TAO CT10中的每个人都在数据流问题上进行了互动，但是没有新的CNE流程成立了。这主要是因为CT10没有嵌入TAO的需求生成过程中。TAO数据流要求的额外透明度将大大简化此问题。
（S // SI // REL USA，FVEY）另一个可以帮助其他系统使用TAO数据的问题是CNE数据的通用标记和分类。在FOXACID的情况下，文件上没有本地分类标记，并且在将数据发到CT10等外部客户之前，TAO必须剥离分区，目标以及FISA数据。随着CT10获得PL3认证，CT10摄取完整的FOXACID记录变得至关重要。遗憾的是，每个文件缺乏独特的标记和分类会带来额外的挑战，使围绕CNE数据的监督，分类和政策变得复杂。此外，与TAO的进一步讨论表明，这也可能是其他数据流的问题。向TAO数据馈送添加分类标记将极大地帮助TAO数据的所有消费者维持适当的安全程序。
（U // FOUO）TAO工程师致力于创建一个RDF数据库系统，该系统可以存储所有信息，并将其提供给外部客户。随着这种能力的发展，CT10可以从中受益匪浅成为这些数据的消费者。该系统将解决若干问题，包括前一段所述的标记和分类挑战 。
（S // SI // REL USA，FVEY）总之，未来的方向是解决这些现有挑战，以便有效推动更多TAO数据近乎实时地流向分析师。CT10还应与TAO开发和部署RDF数据库的努力保持同步。最后，CT10需要实现数据模型扩展，并将任何当前摄取的TAO数据移动到新模型中。
